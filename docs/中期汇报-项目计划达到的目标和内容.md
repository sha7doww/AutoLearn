# 项目计划达到的目标和内容

## 一、项目总体目标

本项目旨在构建**基于知识追踪的智能学习路径推荐系统**，在继承原"知识图谱智能导学小助手"项目可视化展示功能的基础上，增加智能推荐和个性化学习路径规划能力，帮助学生更科学地规划课程学习顺序。

### 核心技术方法

本项目采用 **Agentic Search + Knowledge Tracking (KT)** 的技术路线：

- **知识追踪（KT）**：通过分析学生的历史成绩和学习行为，建立学生的知识状态模型，量化学生对各个知识点的掌握程度。基于IRT（项目反应理论）模型，将学生的知识掌握情况表示为向量，为个性化推荐提供数据基础。

- **智能体搜索（Agentic Search）**：构建基于LLM的智能Agent系统，能够理解学生的学习需求，自主调用多种工具（知识图谱查询、课程检索、学生状态分析等）进行多步推理，生成个性化的学习路径建议。

- **两者结合**：知识追踪提供学生的"知识画像"，智能体搜索根据这个画像动态调整推荐策略——基础薄弱的学生会获得更详细的先修课程建议和学习方法指导，基础扎实的学生则直接推荐进阶内容。

### 解决的痛点

1. **静态推荐问题**：传统系统对所有学生推荐相同内容，本项目根据学生知识状态提供个性化建议
2. **缺乏可解释性**：传统推荐算法像"黑盒"，本项目的Agent能够解释推荐理由（如"因为你在X知识点掌握较好，可以学习Y课程"）
3. **单次推荐局限**：传统系统推荐一次就结束，本项目通过知识追踪持续更新学生状态，动态调整推荐
4. **缺少学习方法指导**：传统系统只告诉学生"学什么"，本项目还能建议"怎么学"

项目的核心目标是：在保留原有3D知识图谱可视化的基础上，建立完整的前后端架构，实现基于学生知识状态的个性化课程推荐功能，并通过智能对话提供学习指导。

## 二、项目主要内容

### 2.1 智能推荐功能（核心）

**基于Agentic Search + KT的个性化推荐**：
- 实现基于学生知识状态的课程推荐算法
- 根据学生当前的知识掌握情况，推荐合适的后续课程
- 考虑课程先修关系，避免推荐学生无法胜任的课程
- 提供推荐理由和学习路径规划，增强系统可解释性
- 支持推荐结果的动态调整：随着学生学习进展，自动更新推荐内容

### 2.2 知识追踪模块

- 基于IRT（项目反应理论）模型，从学生的历史成绩推断其知识掌握状态
- 将学生的知识掌握情况表示为向量形式，便于后续分析
- 支持从课程成绩和练习记录两方面更新知识状态
- 提供知识状态可视化，让学生了解自己的强项和弱项
- 为个性化推荐提供数据基础

### 2.3 智能体（Agent）系统

- 基于LangChain框架构建对话式Agent
- Agent可以调用多个工具：知识图谱查询、课程信息检索、学生状态查询等
- 支持多轮对话，理解学生的学习目标和困惑
- 提供自然语言交互，降低使用门槛
- 实现推理-行动循环（ReAct），进行多步推理生成学习建议

### 2.4 知识图谱构建

- 构建涵盖计算机科学专业主要课程的知识图谱（50-100门课程）
- 标注课程之间的先修关系和知识点依赖
- 使用Neo4j图数据库存储，支持复杂关系查询
- 为每门课程标注难度、学分、主要知识点等属性

### 2.5 系统架构优化

**前端层改进**：
- 保留原有Vue 3 + Three.js的3D知识图谱可视化
- 优化数据获取方式，从后端API动态加载课程数据
- 改进智能对话界面，提供更流畅的交互体验
- 新增学习路径展示组件，可视化推荐的学习顺序

**后端系统建设**：
- 搭建FastAPI后端框架，提供RESTful API服务
- 实现用户认证和会话管理
- 建立完整的数据库系统，规范化数据存储

### 2.6 数据存储系统

- **Neo4j**：存储课程知识图谱和依赖关系
- **PostgreSQL**：存储用户信息、成绩数据、学习记录
- **Milvus（可选）**：存储课程描述的向量表示，支持语义检索
- **Redis**：缓存常用查询结果，提升系统响应速度

## 四、与原项目的关系

本项目是对原"知识图谱智能导学小助手"项目的延续和深化：

| 方面 | 原项目 | 本项目 |
|------|--------|--------|
| 前端展示 | 3D知识图谱可视化 | 保留并优化，新增推荐展示 |
| 后端系统 | 无 | 完整的后端架构 |
| 数据管理 | 前端硬编码 | 数据库规范化存储 |
| 智能功能 | 仅聊天界面 | 知识追踪 + 智能推荐 |
| 个性化 | 无 | 基于学生状态的个性化建议 |

本项目在继承原有成果的基础上，补足了技术架构和智能算法方面的不足，使系统从"展示工具"升级为"智慧学伴"。
